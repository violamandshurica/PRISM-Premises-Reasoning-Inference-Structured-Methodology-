# PRISM — Premises Reasoning Inference Structured Methodology

**v2.5 · 単一プロンプト · 単一LLM · 単一パス**

> フォワードパスの制約内で、人間の批判的思考プロセスを模倣する推論scaffoldプロンプト。

---

## PRISMとは

PRISMは単一LLM・単一パス運用を想定したシステムプロンプトです。人間の批判的思考プロセスを模倣した5フェーズ構造を採用しています。

```
Phase 1 — Premises（前提）    : 明示的・暗黙的前提を分離し、矛盾・曖昧性を特定する
Phase 2 — Core Question（中核問い）: 問題を1段抽象化し、真の問いを再定義する
Phase 3 — Decompose（分解推論）: Cause → Mechanism → Outcome の形式で段階的に推論する
Phase 4 — Counterarguments（反論）: 反論を検討し、Phase 1との整合性を確認する
Phase 5 — Conclusion（結論）  : 最終結論を述べ、不確実性があれば明示する
```

目標は「正答率の向上」ではなく、単一フォワードパスという制約内での**失敗モード制御**です。具体的には、誤差伝搬・無批判な前提採用・構造逸脱を抑制します。

---

## 設計思想

**制約を前提にした設計。**
単一パスのLLMは、出力が長くなるほど後半の命令を無視する傾向があります。PRISMはフェーズ数・命令密度・トークン総量のバランスを最適化しています。フェーズを増やすと構造的完全性は理論上高まりますが、実効的な命令保持率は下がります。

**結果よりプロセスの透明性。**
PRISMとバニラ出力の差は、最終的な答えではなく「その答えにどう至ったか」の追跡可能性にあります。推論チェーンが可視化されているため、検証・批判・再現が可能になります。

**反論検討の強制。**
Phase 4はスキップ不可です。反論検討を省いた回答は「丁寧さが足りない」のではなく、構造的に不完全と判断されます。これが確証バイアスを表面化させる主要なメカニズムです。

## 免責事項（使用前にお読みください）

PRISM v2.5は、LLMの本質的制約（次トークン予測・因果モデルの欠如・メタ認知の欠如）をプロンプト構造で緩和する試みです。

記号論理レベルの矛盾検出と目標追跡の改善に効果がありますが、以下のことは**できません**：

- 真の推論・因果操作
- 確率的にcalibratedな確信度の提示
- 5段階以上の連鎖推論における信頼性の保証
- 反事実的因果判断の正確な処理

5ステップ以上の連鎖推論では誤差が顕著になります。出力の利用はその前提で行ってください。
